rewind is a deterministic record/replay debugger for LLM agents. Record a run once; replay it forever without a network or an API key. Fork any step to compute the future that would have happened.

The problem

Reproducing an agent failure today means re-running every step, paying for every token, and hoping the model takes the same path. It won't. Observability tools show you what happened — rewind lets you run it again.

Replay, don't re-run

Recorded responses are served straight from a local SQLite journal — including exact SSE chunk boundaries. The real SDK consumes replayed streams without knowing the difference. No network. No key. No cost.

Fork the timeline

What if the model had said something else? Rewrite any recorded response and replay the edit. Hybrid replay serves history from the journal and goes live only past the divergence — once.

Diff two futures

rewind diff compares runs event-by-event and pinpoints the first divergence — including the subtle case of same request, different response.

Closed under replay

Every run in the journal — original or forked — replays fully offline. Hybrid replay journals everything into a new run, so counterfactuals are as replayable as the original.

Why this is hard

"Same request" has no obvious definition

Bodies serialize with unstable key order; headers carry per-attempt noise. rewind canonicalizes and fingerprints requests with a volatile-header blocklist — chosen so a wrong fingerprint fails loud (replay miss) instead of lying (silent corruption).

Concurrency breaks naive matching

When identical requests race, journal order is completion order — but each caller needs their response back. rewind records arrival order separately and pairs by it, keeping transcripts byte-exact under races.

Streams are not bodies

SDK parsers are sensitive to SSE framing, so rewind records and replays the exact chunk boundaries — and re-frames edited responses into valid SSE so the SDK can still parse a forked stream.

No proxy, no monkey-patching

rewind injects a recording fetch at the SDK transport boundary. No MITM certificates, no global patches, no traffic outside your process.

Quick start

The whole integration is two lines.

import Anthropic from "@anthropic-ai/sdk";
import { fromEnv } from "rewind";

const session = fromEnv();
const client = new Anthropic({ fetch: session.fetch });

// wrap other non-determinism (tools, time, randomness):
const rate = await session.io("fx_rate", fetchRate);

Then drive everything from the CLI:

  • rewind record -- node agent.jsrun live, journal every LLM call, tool result, and io
  • rewind replay <run> -- node agent.jsre-run offline: recorded responses, no key needed
  • rewind fork <run> --at 46 --text …rewrite step 46, compute the alternate future
  • rewind diff <run> <fork>see exactly where and how the futures diverge
  • rewind step <run>walk the timeline interactively, event by event

The guarantee, stated precisely

Replay is deterministic at the SDK I/O boundary. If your agent's only non-determinism flows through the wrapped client and io(), replay reproduces recorded values byte-exact — including which racing caller got which response. rewind does not (and does not claim to) control instruction-level or OS-level non-determinism.
